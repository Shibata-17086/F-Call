F-Call サーバー管理画面

【サーバー操作時の動作説明】

  • サーバーの起動（node server.js）
    サーバーが起動し、全ての画面（受付・スタッフ・待合室・管理）とリアルタイムでデータをやりとりできるようになります。
    起動直後は全てのリスト・番号が初期化（空）状態です。
  • サーバーの停止（Ctrl+Cやpkill -f server.js）
    サーバーが停止し、全ての画面との通信が切断されます。
    サーバーが停止中は、どの画面でも番号の発券や呼び出しができません。
  • サーバーのリセット（管理画面やスタッフ画面の「リセット」ボタン、またはサーバー再起動）
    発券中リスト・発券履歴・呼び出し履歴・現在の呼び出し番号がすべて初期化されます。
    受付番号は1から再スタートします。
  • 受付画面で「受付番号を発行」ボタン
    サーバーに新しい番号の発券をリクエストします。
    サーバーは「今まで発券した最大番号+1」を新しい番号として発券し、発券中リスト・発券履歴に追加します。
    すべての画面に即時反映されます。
  • スタッフ画面で発券中番号をクリック
    サーバーに「呼び出しリクエスト」を送信します。
    サーバーはその番号を「現在の呼び出し番号」として記録し、発券中リストから削除、呼び出し履歴に追加します。
    待合室画面の「現在呼び出し中の番号」も更新されます。
  • 管理画面で各種クリア・変更ボタン
    発券中リスト・発券履歴・呼び出し履歴のクリアや、呼び出し番号の変更、全体リセットができます。
    すべての画面に即時反映されます。
  • 依存パッケージ未インストール時
    サーバー起動時に「Cannot find module 'express'」等のエラーが出て動作しません。
    npm install または npm install express socket.io cors を実行してください。
